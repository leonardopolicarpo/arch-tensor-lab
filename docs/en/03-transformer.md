# 03: The Transformer

> 🌐 **Available in other languages:** [Português (Brasil)](../pt/03-transformer.md)

## 1. Overview:

The **LanguageModel** class is the mathematical heart of the Small Language Model (SLM). It inherits from **torch.nn.Module**, which is the base class in PyTorch for building any neural network.

* _\_\_init\_\__: I declare the weight matrices, namely the *Embeddings* table and the *Linear* layer, and allocate memory space, preparing the infrastructure but not processing data.
By calling the parent class initializer (**nn.Module**), the model is registered, allowing the framework to track parameters internally, build the computational graph for calculating derivatives (Autograd) during training, and enable the export of final weights (**\.pt**) for future use.

* _forward_: the mandatory route through which data tensors flow, layer by layer, until they are transformed into a final prediction.

## 2. The Constructor

The 'physics' of the model is created here, allocating in memory the two mathematical structures that will translate what the network "reads", "thinks", and "speaks".

### 2.1. **nn.Embedding** (The meaning table)

* The problem: Neural networks do not understand letters or numeric IDs. If I pass ID 4 and ID 2 to the network, the math will mistakenly assume that one is worth twice as much as the other. They are just categories, not magnitudes. I need to transform categories into spatial coordinates.

* The solution: **nn.Embedding** creates a high-speed search table (a Lookup Table). It is allocated in RAM, where it has **V** rows (where **V** is the *vocabulary_size*, e.g., 62) and **D** columns (the *embedding_dimension*, e.g., 128).

* When PyTorch receives the token with ID 5, it doesn't perform matrix multiplications, but instead goes directly to row 5 of this table and fetches an entire vector containing 128 decimal numbers. These 128 numbers are the coordinates of that token in the AI's reasoning space. This is how the network abstracts the 'meaning' of a character.

### 2.2. **nn.Linear** (The prediction head / LM Head)

* The problem: The model was configured to 'think' using 128 dimensions (**D**), but I need it to give me a multiple-choice answer based on the 62 vocabulary options (**V**).

* The solution: The **nn.Linear** layer (*language_modeling_head*) acts as a translator / kind of funnel, applying a linear transformation ($y = xA^T + b$) to project the internal vector space back into the vocabulary space.

* It receives the 128-dimensional vector generated by the Embedding and transforms it into 62 numbers. These are called *logits*, and they represent the 'guessing strength' (the score) of the AI for each possible character. The one with the highest score is the prediction for the next letter in the sequence.

## 3. The Data Flow (**forward**) and Error Calculation

The **forward** method receives the raw IDs and transforms them into predictions, calculating how far the AI is from the correct answer.

### 3.1. The Tensor's Journey (the outward path)

1. Input: the method receives *input_indices* (**X** context), with the format of this two-dimensional matrix being: **[batch_size, sequence_length]** (e.g., **[32, 64]**)

2. Gaining depth: By passing **X** through the *token_embedding_table*, each simple number becomes a 128-dimensional vector. The tensor transforms into a 3D cube with the format **[batch_size, sequence_length, embedding_dimension]**

3. 'Funneling' for the answer: This 'cube' passes through the *Linear* layer (**language_modeling_head**), where it compresses this dimension back to the vocabulary size (62). The final format of the prediction scores (**logits**) is **[batch_size, sequence_length, vocabulary_size]**

### 3.2. The Loss Mathematics (loss calculation)

If I pass the **targets** matrix (the **Y** ground truth) to the **forward**, the network will calculate its 'test score', which is called Loss.

- The problem: PyTorch's evaluation function requires a continuous two-dimensional list, and our matrices were generated as 3D **[B, T, C]**

- The solution: I use the *view()* method to reshape the tensor without altering the data in memory, taking the 32 command sequences (each 64 characters in length) and stacking them all into a single continuous queue.
  - *logits* becomes **[batch_size * sequence_length, channels]** -> **[2048, 62]**
  - *targets* becomes **[batch_size * sequence_length]** -> **[2048]**

- *cross_entropy*: with these matrices, PyTorch compares the 62 probabilities generated by the network with the correct answer and returns a scalar number (e.g., 4.24). The lower this number, the smarter the network is getting, whereas a high *Loss* means the network is guessing randomly.
